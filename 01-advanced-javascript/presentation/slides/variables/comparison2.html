#1: Variables

JavaScript is dynamically typed and "autocasts" types to compare. This can lead to funny behavior, for example:


  0 == "0"; // true
  0 == ""; // true
BUT: 

  "0" == ""; // false

Always use ===

  • 
  "1" === 1; // false
        
  • 
  [] === ""; // false
        
  • 
  0 === "0"; // false
        
  • 
  0 === ""; // false

  • Een voorbeeld van zo'n onverwacht resultaat is het volgende
  • De eerste twee expressies geven beiden true
  • Een logische redenering zou zijn dat als zowel "0" en "" gelijk zijn aan 0, dat ze onderling ook gelijk zijn
  • Dit is niet het geval!
  • Gebruik daarom altijd triple comparison (drie is-gelijk-aan tekens)
  • De triple comparison operator geeft enkel true als de vergelijking klopt en de types hetzelfde zijn
  • Alle expressies die tot nu toe true gaven, zijn nu false
