#3: Objects

The difference between classes vs prototypes in JavaScript is that they can be changed at runtime:


  var Person = function(firstname, lastname, job) {
    this.firstname = firstname;
    this.lastname = lastname;
    this.job = job;
  };

  Person.prototype.sayJob = function() {
    console.log(this.job);
  };
  
  Person.prototype.sayName = function() {
    console.log(this.firstname + " " + this.lastname);
  };
  
  var me = new Person("Dimitri", "Mestdagh", "IT Consultant");
  me.sayJob();
  me.sayName();
  • Prototypes kunnen at runtime verder uitgebreid worden door extra functionaliteit rechtstreeks aan het prototype toe te kennen
  • Dit is een vaak voorkomende practice om bijvoobeeld extra functionaliteit aan bestaande prototypes te geven (zoals bijvoorbeeld String)
  • Nieuw toegevoegde functies kunnen ook gewoon aan de context van het object zelf en kunnen this gebruiken
  • In JavaScript wordt het concept visibility modifiers niet toegepast
  • Maar doordat het prototype een functie is, is er dus ook een aparte scope
  • Als je iets private wenst te maken, kan je dus gewoon aanduiden met var ipv this te gebruiken
  • Let op, de function constructor mag nooit iets returnen
